1. Identidad

Xoria es una asistente operativa con memoria, criterio y capacidad de ejecución.
No nació para conversar por conversar. Nació para reducir carga mental, ordenar contexto, decidir bajo reglas y empujar trabajo real hacia adelante.

En el mercado, la marca visible será Xoria.
En el backend, el motor seguirá existiendo como AL-E Core, en honor al origen del sistema y como núcleo interno de inteligencia, memoria y ejecución.

Xoria no compite por ser “otro chat con IA”.

### BRIEF TÉCNICO PARA FRONT — SOPORTE BILINGÜE ES/EN
**Fecha:** 15 marzo 2026 | **Prioridad:** P1

---

#### Qué hizo Core

Se implementó soporte bilingüe completo en el backend:
- Nueva columna `preferred_language` en `user_profiles` (ya en BD — migración aplicada)
- El orquestador inyecta el idioma en cada conversación automáticamente
- La voz TTS cambia de acento según el idioma del usuario
- El chat libre detecta el idioma del mensaje y responde en consecuencia

---

#### Qué necesita Front implementar

**1. En la pantalla de Perfil / Configuración del usuario**

Agregar un selector de idioma que llame:

```
PATCH /api/profile/me
Authorization: Bearer <token>

Body: { "preferred_language": "en" }   ← o "es"
```

El endpoint ya valida que solo acepte `"es"` o `"en"`. Todo lo demás retorna error 400.

**2. Leer el idioma al cargar el perfil**

```
GET /api/profile/me
```

La respuesta ya incluye `preferred_language` en el objeto de perfil. Úsalo para:
- Pre-seleccionar la opción en el selector de idioma
- Aplicar el idioma de la UI (labels, fechas, textos estáticos)

**3. UI del selector (sugerencia)**

```
🌐 Idioma / Language
  ○ Español (México)   ← valor: "es"
  ○ English (US)       ← valor: "en"
```

Un radio button o toggle es suficiente. No hay más opciones por ahora.

---

#### Comportamiento esperado (para testing)

| Escenario | Resultado |
|---|---|
| Usuario configura `en`, abre chat | Asistente responde en inglés |
| Usuario con `en` escribe en español | Asistente responde en español ese mensaje, luego vuelve a inglés |
| Usuario configura `es` | Comportamiento actual sin cambios |
| Voz con usuario `en` | Escucha voz `en-US-JennyNeural` (acento americano) |
| Voz con usuario `es` | Escucha voz `es-MX-DaliaNeural` (acento mexicano) |

---

#### Lo que Front NO necesita hacer

- ❌ No traducir strings del backend — el asistente ya habla en el idioma correcto
- ❌ No manejar idioma en el chat — el orquestador lo hace automáticamente
- ❌ No tocar nada de voz — Core selecciona la voz según el perfil

Solo: **selector en perfil + leer el campo + aplicar idioma a la UI estática**.

## 29. Idioma y voz natural

### Soporte bilingüe (Español / English)

Xoria es un producto para mercados hispanohablantes e inglés.
El idioma no es una preferencia cosmética — es parte de la experiencia operativa del usuario.

**Regla de idioma:**

El usuario configura su idioma preferido en su perfil (`preferred_language: 'es' | 'en'`).
Toda respuesta del asistente, formatos de fecha y etiquetas del sistema deben respetar ese idioma.

**Regla de chat libre:**

Si en una conversación el usuario escribe en el otro idioma (por ejemplo, su perfil es español pero escribe en inglés en un mensaje), el asistente responde en el idioma de ese mensaje.
Luego vuelve al idioma configurado en el perfil.
No requiere que el usuario cambie su configuración.

**Valores válidos:**
- `es` — Español mexicano (default)
- `en` — English (US)

**Voces TTS por idioma:**

| Idioma | Voz mujer | Voz hombre |
|---|---|---|
| Español | `es-MX-DaliaNeural` | `es-MX-JorgeNeural` |
| English | `en-US-JennyNeural` | `en-US-GuyNeural` |

Proveedor: Microsoft edge-tts (sin costo, sin API key).

**Principio:**

Xoria no habla un solo idioma porque sus usuarios tampoco.
La configuración es del usuario, no del producto.

### Voz natural (TTS)

La síntesis de voz debe sonar como una persona hablando, no como un lector de pantalla.

**Reglas de preprocesamiento antes de sintetizar:**
- Eliminar emojis, markdown, bullets, separadores visuales (`═══`, `───`)
- Convertir saltos de línea en pausas naturales (coma o punto)
- Máximo 3 oraciones por respuesta en modo voz
- Siglas problemáticas reescritas fonéticamente (`API` → `a-pi`)

**Principio:**

La voz es interfaz. La calidad de la voz depende más de lo que se le da que del motor que la sintetiza.

## ⚠️ Reglas de ejecución — AL-E Core (macOS/zsh)

### ❌ NO usar `tsx`
`tsx` falla con `EPERM` por IPC en este entorno. **Nunca** usar:
```
npx tsx ...
```

### ✅ Comandos correctos

**Build (TypeScript):**
```
npm run build
```

**Correr tests P0 de TruthLayer:**
```
node -r ts-node/register scripts/test-truthlayer.p0.ts
```

**Activar TruthLayer (runtime):**
```
export TRUTH_LAYER_ENABLED=true
```

**Rollback inmediato:**
```
export TRUTH_LAYER_ENABLED=false
```

**Flujo completo:**
```
export TRUTH_LAYER_ENABLED=true
npm run build && node -r ts-node/register scripts/test-truthlayer.p0.ts
```
Y si van a hacer smoke en runtime:

export TRUTH_LAYER_ENABLED=true (o rollback false).
